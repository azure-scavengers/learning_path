# Server Hardware Training Program

## Overview

This 7-day training plan is designed for new joiners, moving from **beginner fundamentals** to **expert-level decision-making** in server hardware.

- **Total class time:** 12 hours (max)
- **Audience:** Freshers, junior engineers, IT support, platform trainees
- **Goal:** Build practical confidence in selecting, operating, troubleshooting, and scaling server hardware

---

## Learning Path (Beginner to Expert)

| Day | Duration | Level | Module | Core Topics | Hands-on |
|---|---:|---|---|---|---|
| 1 | 1.5h | Beginner | Server Basics | What is a server, form factors (tower/rack/blade), on-prem vs cloud hardware view | Identify hardware parts from sample diagrams/photos |
| 2 | 1.5h | Beginner | Core Components | CPU, RAM, motherboard, PSU, NIC, storage interfaces (SATA/SAS/NVMe) | Component matching exercise |
| 3 | 1.5h | Intermediate | Storage & RAID | HDD vs SSD vs NVMe, RAID 0/1/5/6/10, controller basics, hot swap | RAID design scenarios |
| 4 | 1.5h | Intermediate | Performance & Sizing | CPU core/thread planning, memory channels, storage IOPS, bottleneck basics | Sizing worksheet for common workloads |
| 5 | 2.0h | Advanced | Reliability & Operations | Redundancy (PSU/NIC/fans), thermal design, firmware/BIOS/iDRAC/iLO, monitoring essentials | Failure-mode drill and response planning |
| 6 | 2.0h | Advanced | Security & Compliance | Secure boot, TPM, firmware hardening, physical security, hardware lifecycle governance | Hardening checklist walkthrough |
| 7 | 2.0h | Expert | Architecture & Cost Optimization | Build-vs-buy, refresh strategy, workload placement, TCO, capacity planning | Capstone: propose hardware architecture for a business case |

**Total = 12 hours**

---

## Module Outcomes

By the end of the program, participants will be able to:

1. Explain server hardware fundamentals clearly.
2. Choose suitable hardware configurations for common workloads.
3. Apply reliability and security best practices.
4. Diagnose basic performance and hardware issues.
5. Present a justified architecture recommendation with cost-performance trade-offs.

---

## Delivery Format

- **Session model:** 60% instructor-led, 40% practical discussion/labs
- **Daily flow:** recap (10 min), core module, activity/lab, Q&A
- **Suggested batch size:** 15 to 25 learners

---

## Assessment Plan

- **Daily quick quiz:** 5 to 8 questions
- **Scenario-based tasks:** Days 3 to 6
- **Final capstone review:** Day 7
- **Completion criteria:** 70%+ score + capstone participation

---

## Materials Checklist

- Slide deck per day
- Component reference sheet (CPU/RAM/storage/network)
- RAID and sizing cheat sheets
- Monitoring and hardening checklists
- Capstone template (requirements, assumptions, recommendation, risks)

---

## Suggested Capstone Prompt

Design server hardware for a mid-size enterprise app stack:
- Web tier, API tier, database tier
- High availability required
- 20% yearly growth expected
- Budget-constrained, but no single point of failure allowed

Deliverables:
- Proposed hardware spec
- Redundancy plan
- Security controls
- Cost and scalability rationale
